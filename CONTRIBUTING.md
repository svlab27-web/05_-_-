# Руководство по участию в проекте

## Введение

Мы рады, что вы хотите внести свой вклад в наш проект! Это руководство поможет вам эффективно участвовать в разработке.

## Процесс участия

1. **Форкните репозиторий**
   - Создайте fork проекта на GitHub
   - Клонируйте свой fork локально

2. **Создайте ветку для разработки**
   ```bash
   git checkout -b feature/название-фичи
   # или
   git checkout -b bugfix/описание-бага
   ```

## Стандарты разработки

### Код
- Используйте Python 3.8+
- Следуйте стандартам PEP 8
- Используйте type hints
- Пишите документацию для функций и классов
- Покрывайте код тестами

### Commits
- Используйте семантические commits
- Префиксы:
  - `feat:` - новая функциональность
  - `fix:` - исправление бага
  - `docs:` - изменения в документации
  - `refactor:` - рефакторинг кода
  - `test:` - добавление/изменение тестов
  - `chore:` - обслуживание проекта

### Pull Requests
- Используйте шаблон PR
- Описывайте суть изменений
- Убедитесь, что все тесты проходят
- Получите одобрение от maintainer'ов

## Локальная разработка

1. Установите зависимости:
```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
pip install -r requirements-dev.txt
```

2. Запуск тестов:
```bash
pytest tests/
```

3. Проверка линтеров:
```bash
flake8 src/
mypy src/
```

## Отчеты о проблемах

- Используйте шаблоны Issues
- Предоставляйте максимально подробную информацию
- Прикладывайте логи и скриншоты

## Кодекс поведения

- Будьте вежливы и уважительны
- Конструктивно обсуждайте идеи
- Помогайте друг другу

## Лицензия

Участвуя в проекте, вы соглашаетесь с условиями лицензии MIT.

## Вопросы?

Если у вас есть вопросы, создайте Issue или свяжитесь с maintainer'ами.
